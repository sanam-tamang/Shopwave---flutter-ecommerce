The LayoutScaffold widget, which handles BottomNavigationBar with GoRouter, should be placed in a "widget" folder as a reusable widget. It acts as a layout wrapper, meaning multiple pages/features will use it.

Recommended Folder Structure

bash
lib/
 core/
    router/
       app_router.dart  # Defines GoRouter routes
 features/
    home/
       presentation/
          home_page.dart
    profile/
       presentation/
          profile_page.dart
 widgets/
    layout_scaffold.dart  # Handles BottomNavigationBar + GoRouter
 main.dart

Why Place LayoutScaffold in widgets/?

  • Reusability: Multiple features (home, profile, settings) use the same bottom navigation.
  • Separation of Concerns: Feature pages focus on UI logic, while navigation logic remains in LayoutScaffold.
  • Cleaner Codebase: Keeps GoRouter-related navigation in LayoutScaffold, avoiding duplicate code in pages.

How to Use It in Pages?

In home_page.dart, profile_page.dart, etc., wrap the content inside LayoutScaffold.

dart
import 'package:flutter/material.dart';
import 'package:your_app/widgets/layout_scaffold.dart';

class HomePage extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return LayoutScaffold(
      child: Center(child: Text("Home Page")),
    );
  }
}

This way, LayoutScaffold manages navigation while each feature focuses on its own content.

The core/ folder holds essential logic that is used across multiple features, such as routing, dependency injection, services, and data management.

Response 1

The issue occurs because the dialog is being pushed as a full-screen route (GoRoute), but dialogs should not be handled like full-screen pages in GoRouter. The dialog does not get dismissed because its being treated as a separate page, rather than a modal overlay.

Solution: Use showDialog() Instead of Defining a Dialog Route

Since you want to show a sign-in dialog instead of navigating to a new page, you should remove the signInWithDialog route from GoRouter and handle it manually using showDialog().

Step 1: Remove This Route

In AppRoute, remove the signInWithDialog route:

dart
GoRoute(
  path: AppRouteName.signInWithDialog.path,
  name: AppRouteName.signInWithDialog,
  builder: (context, state) {
    return Dialog(child: SignInPage()); //  REMOVE THIS
  },
),

Step 2: Show Dialog Manually in _redirect()

Modify the _redirect function to show a dialog manually instead of navigating:

dart
static FutureOr<String?> _redirect(context, state) async {
  final currentPath = state.uri.path;
  final isAuthPath = currentPath == AppRouteName.signIn.path ||
      currentPath == AppRouteName.signUp.path;

  final doesUserNeedToBeAuthenticatedBeforeAccesingCurrentPath =
      currentPath == AppRouteName.userProfile.path ||
          currentPath == AppRouteName.cart.path;

  final failureOrUser = await sl<UserLocalDataRepository>().getData();
  final currentUser = failureOrUser.fold((failure) => null, (user) => user);

  if (doesUserNeedToBeAuthenticatedBeforeAccesingCurrentPath &&
      currentUser == null) {
    //  Instead of navigating, show the dialog manually
    Future.microtask(() {
      showSignInDialog(context);
    });

    //  Stay on the same page instead of redirecting
    return null;
  }

  if (isAuthPath && currentUser != null) {
    return AppRouteName.home.path;
  }
  return null;
}

Step 3: Define showSignInDialog

Put this function inside common/utils/dialogs.dart:

dart
void showSignInDialog(BuildContext context) {

    context: context,

            scale: CurvedAnimation(
              parent: ModalRoute.of(context)!.animation!,
              curve: Curves.easeInOut,

}
